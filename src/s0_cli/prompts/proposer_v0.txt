# Process — follow in this order

1. **Read the seed harness FIRST.** Call
   `read_harness("baseline_v0_agentic")` (and optionally
   `read_harness("baseline_v0_singleshot")`). You must copy the import block,
   the `__init__` plumbing (`_llm`, `_tools`, `_loop`), and `with_no_llm`
   verbatim — the runtime expects these. Skipping this step is the #1 cause
   of broken candidates.
2. **Diagnose.** Read at least 3 traces from prior runs to see what failed.
   Look at `false_positives` and `false_negatives` in `scored.json` per task.
3. **Form a hypothesis** about a specific bottleneck (over-reporting on XSS,
   missing vibe-code patterns, wasted turns on dead-end files, etc.).
4. **Pick a small, additive change**. The paper's section A.2 reports that
   "purely additive" wins after multiple regression iterations. Start from
   the full baseline source and add ONE thing (a post-filter, an extra
   prompt instruction, a new heuristic). Do not rewrite from scratch.
5. **Write the new harness file** with `write_harness`. Choose a descriptive
   name like `v1_xss_filter` or `v1_vibe_aware`. Inherit from `Harness`,
   keep `name = "<name>"` matching the filename.
6. **(Optional)** Write a new prompt template if you want to change the
   system message. Reference it from your harness with `prompts.load("name.txt")`.
7. **Call `validate` to self-check**. The validator parses your file, looks
   for a Harness subclass, checks imports, etc. If it returns `ok: false`,
   read its `errors`, fix the file with another `write_harness`, and call
   `validate` again until it passes.
8. **Call `finish`** with a 2-3 sentence summary of what you changed and the
   expected effect (precision/recall/cost direction).

# Common LLM mistakes to avoid

- Wrapping the entire file in a triple-quoted string (`""" ... """`). The
  validator will see "No class subclassing Harness found." Use plain
  comments (`# ...`) for the file header instead.
- Forgetting `from __future__ import annotations` — `Target` and `ScanResult`
  in the `scan` signature need it on Python 3.11.
- Using `Tools.semgrep` or `Tools.<scanner>`. Tools is instantiated with
  `Tools(ctx)`; scanners are invoked via `Tools.run_scanner` (a tool method
  inside the agent loop) or by importing `SemgrepScanner` from
  `s0_cli.scanners.semgrep` directly and calling `.run(target)`.
- Returning a dict instead of a `ScanResult` instance.
